# NoSQL
* Not Only SQL，指的是非关系型数据库
* 因为关系型数据库，随着访问量上升，硬盘读取数据就出现了性能问题，为了解决这个问题，内存级的读写nosql就产生了。

### 优缺点
* 优点：
  1. 没有复杂关系
  2. 可扩展性高
  3. 分布式计算
  4. 架构灵活性，半结构化数据
* 缺点：
  1. 没有标准化（不像关系数据库基于ER模型）
  2. 有限的查询功能

### 分类
* 列存储：Hbase...
* 文档型存储：[mongodb](./mongodb/mongodb01.md)...
* 键值对存储：[redis](./redis/redis01.md)...
